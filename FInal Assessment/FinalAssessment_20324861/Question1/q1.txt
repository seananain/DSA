Sorting an 800 element array for both mergeSort and rightmost pivot quickSort will yield very different
results. As shown above, for an  unsorted 8 element integer array, mergeSort has a total function call 
number for mergeSortRecurse as 7 and a recursion depth of 3. For quickSort, however, with an already sorted
8 integer array, quickSortRecurse is called 13 times, with a recursion depth of 7. So for an 800 element 
array, it can be expected to see a much worse performance for quickSort compared to mergeSort.
